> DataBase:

A database is a place where we store data in an organized way, so we can easily create, read, update, and delete that data whenever we needed.
Its like a digital filing cabinet for information.

> DBMS:

A DBMS (Database Management System) is software acts as a bridge between the user and the database that helps to manage the database
and it supports CRUDS operations, ensuring data is secure and organized.

> DBMS types:

1. SQL:

SQL stands for Structured Query Language. It is used to work with relational databases.
Here data is stored in tables with rows and columns, and the structure is fixed using a schema.
Because of this structured format, SQL is good for applications like banking or billing systems where data has clear relationships.
Examples are MySQL, PostgreSQL, and Oracle.

2. NoSQL:

NoSQL databases are non-relational and flexible. They store data as documents or key-value pairs.
They are schema-less, so each doc can have different fields, and we can change structure anytime.
Because of this flexibility, NoSQL is used for large scalable apps.
NoSQL means ‘Not Only SQL’, so it doesn't always use SQL commands.
Examples: MongoDB and Firebase, couchDB

when to use:

Use SQL when data is structured, fixed, and has clear relationships like banking and billing.
Use NoSQL when data is growing fast, and needs scalability like social media.
