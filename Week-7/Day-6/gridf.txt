> GridFS:

GridFS is used to store very large files in MongoDB.
MongoDB has a document size limit (16MB), so we cannot store big files directly inside one document.
GridFS solves this by splitting the file into small pieces called chunks.
Each chunk is stored separately in the database.
When we retrieve the file, MongoDB automatically combines all the chunks and returns the full file.

How MongoDB stores it internally:

Internally, MongoDB automatically creates two collections:
fs.files → stores file details like file name, size, upload date, etc.
fs.chunks → stores the actual file data in small chunks.
The default chunk size is 255 KB.
So if a file is large, MongoDB divides it into 255 KB chunk document and stores them in fs.chunks.
