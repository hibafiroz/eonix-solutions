Change Streams:

- Change Streams are used to get real-time updates from MongoDB collections.
- Whenever there is an insert, update, or delete, MongoDB automatically sends a change event to our application, so we don’t need polling.
- It work only in Replica Set or Sharded Cluster, not standalone, bcz they depend on MongoDB’s oplog.
- They are mainly used in chat apps, notifications, and live dashboards.

To use it, we call:
const changeStream = db.users.watch();

Then listen like:
changeStream.on("change", change => printjson(change))

If we want only inserts field, we use $match:
db.users.watch([{ $match: { operationType: "insert" }}])

And if we want the full updated document after update:
db.users.watch([], { fullDocument: "updateLookup" })
