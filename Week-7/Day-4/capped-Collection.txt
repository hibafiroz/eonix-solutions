Capped Collections:

- A capped collection is a fixed-size collection.
  It is basically used when we want to keep only the latest data. 

- We define a fixed size for the collection 
  and Once it reaches its size limit, MongoDB doesn't stop inserting data 
  instead it automatically removes the oldest documents to insert new ones.

- It's mainly used for logs or streaming data where we only care about the most recent records.

Example:

db.createCollection("logs", {
  capped: true,
  size: 1024 * 1024,  // 1MB (given in bytes)
  max: 1000    //max documents we can insert (optional)
})
MongoDB is allowed to use ONLY 1MB for this collection


> Characteristics of Capped Collection:

1. It must be explicitly created bcz MongoDB will NOT automatically creates the capped collections.
We also can't convert a normal collection into capped easily. It must be created capped from the start.

2. It is created with Fixed size and that size cannot change it later.

3. When it reaches its size limit, MongoDB automatically overwrites the oldest documents.

4. Documents stay in the exact order they were inserted.

5. We can update fields only if size doesnt grow.

6. We cannot delete document individually also We can't drop entire collection.

7. We cannot use TTL index bcz the deletion happens in TTL and Text index bcz tokenization, inverted index structures, frequent updates
