7. Sparse Index:

- A sparse index indexes only the documents that actually contain the indexed field.
  By default, If a document doesn't have that field, it still creates index as null to that document.
  So to avoid that we use sparse index. This reduces the index size.

> Example:
[
  { name: "Hiba", age: 23 },
  { name: "Sara" },
  { name: "Ayaan", age: 25 }
]

db.users.createIndex({ age: 1 }, { sparse: true })

Only these get indexed:
{ name: "Hiba", age: 23 }
{ name: "Ayaan", age: 25 }

If we run:
db.users.find({ age: null })
Sparse index will not return documents without age bcz they are not indexed at all.
