> useEffect Hook:

useEffect is a React Hook used to handle side effects in functional components. 
Side effects include things like fetching data from an API, setting up event listeners or starting a timer.
useEffect runs after the component renders, and we can control when it runs using the dependency array.

useEffect(() => {
  console.log("Component mounted");
}, []);

- The first part is the function that contains the effect.
- The second part is the dependency array, which controls when the effect should run.


> Cleanup Function:

When the component is removed, things created outside React do not automatically removes.
- event listeners
- setInterval
- setTimeout
- websocket connections

These are attached to the browser, not inside the component itself.
So when the component disappears, those things can still run unless we stop them. 
It can cause memory leaks or duplicate listeners can appear.
That’s why we use the cleanup function inside useEffect.

> Cleanup does two times:
1. Before the effect runs again (when dependencies change)
2. When the component unmounts (component removed)

- Cleaning timer:

useEffect(() => {

  const timer = setInterval(() => {
    console.log("Running...");
  }, 1000);

  return () => {                    // Cleanup function must not return anything except a function.
    clearInterval(timer);
  };

}, []);


- Cleaning Event Listener using useEffect:

function App() {
  useEffect(() => {
    const handleResize = () => {
      console.log("Window resized");
    };

    window.addEventListener("resize", handleResize);

    return () => {
      window.removeEventListener("resize", handleResize);
    };

  }, []);

  return <h1>Resize the window</h1>;
}


> Multiple useEffect in a Component:

Each useEffect should handle one specific task.
Instead of putting everything in one big effect, we separate logic.

import { useEffect, useState } from "react";

function App() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log("Component mounted");
  }, []);

  useEffect(() => {
    console.log("Count changed:", count);
  }, [count]);

  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  );
}
First useEffect:

useEffect(() => {
  console.log("Component mounted");
}, []);
This runs only once when the component loads.

Second useEffect:

useEffect(() => {
  console.log("Count changed:", count);
}, [count]);
This runs every time count changes.

- Execution Order
React runs useEffect in the order they appear in the component.

- In React 18 and React 19 -> In development mode, React may run effects twice automatically to detect bugs.
This happens in Strict Mode. But in production it runs normally.



> Note:

Inside useEffect:
add something → clean it in return()
example:
addEventListener → removeEventListener
setInterval → clearInterval
subscribe → unsubscribe
