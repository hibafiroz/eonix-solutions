> useMemo Hook:

useMemo is used for performance optimization.
If there is a heavy calculation inside a component, normally it will run on every render.
So we use useMemo to store the calculated value and reuse it. It will only recalculate when the dependency changes.
This helps to improve performance.

Difference from normal variable
const value = expensiveFunction(); // runs every render

const value = useMemo(() => expensiveFunction(), []); // runs once


import { useState, useMemo } from "react";

function App() {
  const [count, setCount] = useState(0);
  const [text, setText] = useState("");

  const expensiveCalculation = useMemo(() => {
    console.log("Calculating...");
    return count * 2;
  }, [count]);

  return (
    <div>
      <h1>{expensiveCalculation}</h1>

      <button onClick={() => setCount(count + 1)}>
        Increase Count
      </button>

      <input 
        value={text}
        onChange={(e) => setText(e.target.value)}
      />
    </div>
  );
}

When count changes -> calculation runs
When text changes -> calculation does NOT run
Because useMemo only cares about [count]


> note:

It returns a value, not a function
It runs only when dependencies change
Don’t overuse it (only for heavy stuff)
