> Synthetic Event:

Synthetic Event is basically a wrapper around the browser's native event. 
When a user performs an action, the browser creates a normal event but React wraps that event inside an object called a SyntheticEvent
and passes it to event handler to make sure events work the same in all browsers to improve performance.

Different browsers behave slightly differently like
Event properties may differ
Old browsers behave differently
So developers had to write extra code to support all browsers.

function handleClick(e) {
  console.log(e.type);
}

<button onClick={handleClick}>Click</button>
e = SyntheticEvent

| Property              | Meaning                      |
| --------------------- | ---------------------------- |
| `e.type`              | event type                   |
| `e.target`            | element that triggered event |
| `e.preventDefault()`  | stop default behavior        |
| `e.stopPropagation()` | stop event bubbling          |


> Why React uses Synthetic Events?

1. Cross-browser compatibility
Different browsers handle events differently. SyntheticEvent makes events work the same in all browsers.

2. Better performance
React uses event delegation (one event listener for many elements), which improves performance.

3. Consistent behavior
SyntheticEvent provides the same properties and methods across all browsers.

4. React controls events
React can manage and optimize events efficiently using SyntheticEvent.


> Advantages of Synthetic Events:

1. Cross-Browser Compatibility
Synthetic events work consistently across all browsers.

2. Consistent API
Developers use the same event properties and methods regardless of the browser.

3. Better Performance
React uses event delegation. Instead of attaching many event listeners to individual elements, React attaches a single listener at the root and manages events internally.

4. Efficient Event Management
React controls event handling and optimizes performance.
