> useState Hook:

useState is a React Hook that is used to create and update state inside a functional component.
When the state changes, the component re-renders and the UI updates automatically.

React only re-renders the affected component, not the entire application.

const [state, setState] = useState(initialValue);

state: current state value
setState:  function to update the state
initialValue: starting value of the state


> The useState hook returns an array with two values:

1. The current state value
2. A function to update that state

we use destructuring bcz useState returns an array. 
Array destructuring allows us to store the state value and update function in separate variables easily.
If we dont use Array destructuring, we would have to access values using indexes.

const state = useState(0)
const count = state[0]
const setCount = state[1]
This works, but it is not clean and harder to read.


> Multiple State Variables:
We can create multiple states.

function User() {

  const [name, setName] = useState("Hiba");
  const [age, setAge] = useState(22);

  return (
    <div>
      <h1>{name}</h1>
      <h2>{age}</h2>
    </div>
  );
}

Each state works independently.


> useState with Objects:
State can also store objects.

const [user, setUser] = useState({
  name: "Hiba",
  age: 22
});

Update object state:

setUser({
  ...user,
  age: 23
});

Wrong update: setUser({ age: 23 });
We use spread operator to keep existing values.


> useState with Arrays:

const [items, setItems] = useState([]);

Adding item: setItems([...items, "Apple"]);


> updating state:
Updating state in React means modifying the value of a state variable using the setter function returned by useState. 


> Updating Array State:

const [items, setItems] = useState([]);

Add item: setItems([...items, "Apple"]);

Here we create a new array with the old items plus the new one.


> Multiple Updates:

setCount(count + 1);
setCount(count + 1);
Here we might expect:

0 → 1 → 2

But React actually sees:

setCount(1)
setCount(1)

Because both lines used the same old value (0).

So result becomes:
count = 1

But when updates depend on previous state repeatedly, the safer pattern is:
Correct Way When Doing Multiple Updates
setCount(prev => prev + 1);
setCount(prev => prev + 1);
Now React processes them sequentially:


> State updates are asynchronous bcz React collects multiple state updates and updates them together to improve performance by avoid unnecessary re-rendering.

> We cannot update state directly bcz React will not re-render. We must use the setter function.

> React uses array so we can give any name to the state variable and setter function.
If it was an object, we would have to use fixed property names.

function useState(initialValue) {
  let state = initialValue;
  function setState(newValue) {
    state = newValue;
  }
  return {
    state: state,
    setState: setState
  };
}

We must use the same names: const { state, setState } = useState(0);
We cannot rename like this: const { count, setCount } = useState(0);

Because object destructuring depends on property names.
array destructuring depends on position, not name.
