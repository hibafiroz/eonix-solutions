> Lists and Keys:

In React, lists are used to render multiple elements using map() function. 
Keys are attributes used to identify elements uniquely in a list so React can efficiently update, add, or remove items during the rendering process.

map() creates a new array by transforming each element.


> Basic Example:

function App() {
  const fruits = ["Apple", "Orange", "Mango"];

  return (
    <ul>
      {fruits.map((fruit) => (
        <li>{fruit}</li>
      ))}
    </ul>
  );
}
Output
- Apple
- Orange
- Mango


function App() {
  const fruits = ["Apple", "Orange", "Mango"];

  return (
    <ul>
      {fruits.map((fruit, index) => (
        <li key={index}>{fruit}</li>
      ))}
    </ul>
  );
}
Here:

key={index}
React uses this key to track elements efficiently.


- Keys help React in its reconciliation process.
- React compares Previous Virtual DOM vs New Virtual DOM
- Using keys, React can determine Which item changed, Which item was added, Which item was removed
- Without keys, React may re-render unnecessary elements, reducing performance.


> Example Without Keys:

{items.map(item => (
  <li>{item}</li>
))}

React will show a warning:

Each child in a list should have a unique "key" prop
Because React cannot track items properly.


> Example With Keys:

{items.map(item => (
  <li key={item.id}>{item.name}</li>
))}

Here item.id is unique. React can track elements correctly.


> Using Index as Key:

{items.map((item, index) => (
  <li key={index}>{item}</li>
))}

But React documentation discourages using index as a key bcz

If list order changes, React may update elements incorrectly.
Using index is acceptable only if The list never changes order, Items are not added or remove, Data has no unique IDs



Lists can also render components.

Example:

function User({ name }) {
  return <li>{name}</li>;
}

function App() {
  const users = ["Hiba", "Sara", "Ali"];

  return (
    <ul>
      {users.map((user, index) => (
        <User key={index} name={user} />
      ))}
    </ul>
  );
}
User component is rendered multiple times.
Each component has a unique key.


> Lists with Conditional Rendering:

const users = [
  { id: 1, name: "Hiba", active: true },
  { id: 2, name: "Sara", active: false }
];

{users.map(user =>
  user.active && <p key={user.id}>{user.name}</p>
)}

Only active users are shown.
React uses keys during Virtual DOM diffing.


> Rules for Keys:

1. Keys must be unique among siblings
2. Keys must be stable
3. Keys are not passed as props

Inside a component: props.key ❌
Key is used internally by React.
