> Event Handling:

Event Handling means responding to user actions like click, input change, and form submit using event handler functions.
We write functions and connect them to elements using events like onClick, onChange, and onSubmit.

| HTML                  | React                 |
| --------------------- | --------------------- |
| Uses lowercase events | Uses camelCase events |
| Uses strings          | Uses functions        |

HTML Example
<button onclick="handleClick()">Click</button>
React Example
<button onClick={handleClick}>Click</button>

> Example:

function App() {

  function handleClick() {
    alert("Button clicked!");
  }

  return (
    <button onClick={handleClick}>Click Me</button>
  );
}

export default App;


> Instead of defining a separate function, we can use an arrow function directly inside JSX.

Example:

function App() {
  return (
    <button onClick={() => alert("Button clicked!")}>
      Click Me
    </button>
  );
}


> Updating State with Events:

function Counter() {

  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);
  }

  return (
    <div>
      <h2>{count}</h2>
      <button onClick={handleClick}>Increase</button>
    </div>
  );
}

When the button is clicked:
handleClick runs
setCount(count + 1) updates the state
React re-renders the component
Updated value appears in the UI


> We can pass data to the event handler.

Example:

function App() {

  function greet(name) {
    alert("Hello " + name);
  }

  return (
    <button onClick={() => greet("Hiba")}>
      Greet
    </button>
  );
}

Arrow function allows passing parameters.
greet("Hiba") runs when the button is clicked.


> Event Object in React

React automatically passes an event object to event handler functions.
This event object contains information about the event.

Example:

function App() {

  function handleClick(event) {
    console.log(event);
  }

  return (
    <button onClick={handleClick}>
      Click Me
    </button>
  );
}

The event object includes:
event type
target element etc.


> Sometimes events bubble up to parent elements.
To stop this, use:
event.stopPropagation()

function App() {

  function parentClick() {
    console.log("Parent clicked");
  }

  function childClick(e) {
    e.stopPropagation();
    console.log("Button clicked");
  }

  return (
    <div onClick={parentClick}>
      <button onClick={childClick}>Click</button>
    </div>
  );
}
Without stopPropagation, both parent and child would trigger.


> What is event binding in React?

In class components, event handlers need to be bound to the component instance so that this refers to the correct component.
So it is done using bind(this) inside the constructor. 
Functional components don't require binding because they don’t use this.


> Common React Events:

Mouse Events
| Event         | Description                       |
| ------------- | --------------------------------- |
| onClick       | Triggered when element is clicked |
| onDoubleClick | Double click                      |
| onMouseEnter  | Mouse enters element              |
| onMouseLeave  | Mouse leaves element              |

Keyboard Events:
| Event      | Description                              |
| ---------- | ---------------------------------------- |
| onKeyDown  | Key pressed                              |
| onKeyUp    | Key released                             |
| onKeyPress | Key pressed (deprecated in modern React) |

Form Events:
| Event    | Description         |
| -------- | ------------------- |
| onChange | Input value changes |
| onSubmit | Form submitted      |
| onFocus  | Input focused       |
| onBlur   | Input loses focus   |

function App() {

  function handleChange(e) {
    console.log(e.target.value);
  }

  return (
    <input type="text" onChange={handleChange} />
  );
}
e.target.value gets the input value
