A Controlled Component is a React form element whose value is controlled by React state using the value prop and updated with the onChange event, making React the single source of truth.

This means:

React stores the form data in state

React controls the value displayed in the form field

Every change in the input updates the React state

So the single source of truth is React state, not the HTML element.


In normal HTML forms, the DOM manages the input value.

Example:

<input type="text">
The browser itself keeps track of what the user types.

But in React controlled components:

React stores the value in state

The input field gets the value from state

When the user types, onChange updates the state

React re-renders and updates the input value

Flow:

User types → onChange runs → state updates → React updates UI



Why Controlled Components Are Used
Controlled components give full control of form data to React.

Benefits:

• Easier form validation
• Easier form submission handling
• Easier dynamic UI updates
• Easy to reset or modify input values
• Enables conditional rendering based on input

Example:

Disable button if input is empty.



Basic Syntax
A controlled input has two important things:

1.value
2.onChange

import { useState } from "react";

function App() {
  const [name, setName] = useState("");

  return (
    <div>
      <input
        type="text"
        value={name}
        onChange={(e) => setName(e.target.value)}
      />

      <p>{name}</p>
    </div>
  );
}

export default App;



How It Works (Step by Step)
Initial State

name = ""
User types H

onChange runs
React updates state

setName("H")
React re-renders

value = "H"
Now the input displays H.

Every keystroke repeats this cycle.

Key Characteristics of Controlled Components
1. React State Controls the Input
The input value comes from React state.

Example:

value={name}
The DOM cannot change it directly.

2. onChange Updates State
Whenever user types, React must update the state.

Example:

onChange={(e) => setName(e.target.value)}
e.target.value contains the current input value.

3. Single Source of Truth
The state variable holds the actual data.

Example:

name → real data
input field → just displaying state

React controlled components work with many form elements.

Text Input

const [isChecked, setIsChecked] = useState(false);

<input
  type="checkbox"
  checked={isChecked}
  onChange={(e) => setIsChecked(e.target.checked)}
/>
e.target.checked gives true or false.

Radio Button
<input
  type="radio"
  value="male"
  checked={gender === "male"}
  onChange={(e) => setGender(e.target.value)}
/>
Handling Multiple Inputs
For forms with multiple fields, we often store values in an object.

Example:

import { useState } from "react";

function Form() {

  const [formData, setFormData] = useState({
    name: "",
    email: ""
  });

  function handleChange(e) {
    const { name, value } = e.target;

    setFormData({
      ...formData,
      [name]: value
    });
  }

  return (
    <form>
      <input
        name="name"
        value={formData.name}
        onChange={handleChange}
      />

      <input
        name="email"
        value={formData.email}
        onChange={handleChange}
      />
    </form>
  );
}

export default Form;
Explanation:

name attribute identifies the field
[value] updates correct state property
Controlled Form Submission
Example:

function App() {

  const [email, setEmail] = useState("");

  function handleSubmit(e) {
    e.preventDefault();

    console.log(email);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={email}
        onChange={(e) => setEmail(e.target.value)}
      />

      <button type="submit">Submit</button>
    </form>
  );
}
Important:

e.preventDefault()
Prevents page reload.

Resetting Controlled Inputs
Since the input value comes from state, resetting is easy.

Example:

setName("");
Input becomes empty immediately.

Advantages of Controlled Components
1. Full Control Over Form Data
React manages every input value.

2. Easy Validation
Example:

if(name.length < 3){
  alert("Name too short");
}
3. Dynamic UI
Example:

Disable button if input empty.

<button disabled={!name}>
Submit
</button>
4. Predictable Data Flow
All data stays inside React state.

5. Easy Form Reset
Just reset the state.

Disadvantages
1. More Code
Need state + onChange for every field.

2. Re-render On Every Keystroke
Each change triggers a React re-render.

For very large forms this may affect performance.

Common Mistakes
Mistake 1 — Missing onChange
<input value={name} />
Error:

Read-only input field
Fix:

Add onChange
Mistake 2 — Using defaultValue Instead of value
defaultValue → uncontrolled
value → controlled
Mistake 3 — Forgetting checked for Checkbox
Wrong:

value={isChecked}
Correct:

checked={isChecked}
Controlled vs Uncontrolled Components
Feature	Controlled	Uncontrolled
Data Source	React State	DOM
Input Value	value prop	defaultValue
State Management	React	Browser
Validation	Easy	Hard
Control	Full control	Limited control
Example Controlled:

<input value={name} onChange={handleChange} />
Example Uncontrolled:

<input defaultValue="Hello" />
Real Life Example
Login Form:

function Login() {

  const [email, setEmail] = useState("");
  const [password, setPassword] = useState("");

  function handleSubmit(e){
    e.preventDefault();

    console.log(email, password);
  }

  return (
    <form onSubmit={handleSubmit}>

      <input
        type="email"
        value={email}
        onChange={(e)=>setEmail(e.target.value)}
      />

      <input
        type="password"
        value={password}
        onChange={(e)=>setPassword(e.target.value)}
      />

      <button>Login</button>

    </form>
  );
}
