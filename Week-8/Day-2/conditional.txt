> Conditional rendering:

Conditional rendering means displaying different UI elements based on conditions. 
It uses JavaScript conditions like if/else, ternary operators, and logical AND to control what gets rendered based on data.
For example, if the user is logged in, we show Logout button. If not, we show Login button. This is conditional rendering.


> React supports multiple ways to handle conditions.

Main methods:
- if / else
- Element variables
- Ternary operator
- Logical AND (&&)
- Switch statement
- Conditional rendering with null
- Conditional component rendering
- Conditional rendering with functions


> if / else Statement:

Used outside JSX because JSX cannot directly contain if statements.
function App() {
  const isLoggedIn = true;

  if (isLoggedIn) {
    return <h1>Welcome User</h1>;
  } else {
    return <h1>Please Login</h1>;
  }
}

- Multiple Conditions Using if:

function StatusMessage({ status }) {
  if (status === "loading") {
    return <div>Loading...</div>;
  }

  if (status === "error") {
    return <div>Error occurred!</div>;
  }

  if (status === "success") {
    return <div>Success!</div>;
  }

  return <div>Unknown status</div>;
}

This is called early return pattern.
Early return pattern is a technique where we return from a function as soon as a condition is met instead of using nested if-else.


> Element Variables
Sometimes we store JSX in a variable based on a condition.

Example:

function App() {
  const isLoggedIn = true;
  let message;

  if (isLoggedIn) {
    message = <h1>Welcome back!</h1>;
  } else {
    message = <h1>Please login</h1>;
  }

  return <div>{message}</div>;
}


> Ternary Operator:
The ternary operator is the most used method for conditional rendering.
condition ? trueResult : falseResult

- Example:

function App() {
  const isLoggedIn = true;

  return (
    <div>
      {isLoggedIn ? <h1>Welcome</h1> : <h1>Please Login</h1>}
    </div>
  );
}


> Logical AND Operator (&&):
Used when we want to render something only if condition is true.
condition && expression

- Example:

function App() {
  const hasMessage = true;

  return (
    <div>
      {hasMessage && <p>You have a new message</p>}
    </div>
  );
}


> Switch statement:
If-else is used when we have range conditions or complex logic.
Switch is used when we compare one variable with multiple exact values

import { useState } from "react";

function StatusExample() {
  const [status, setStatus] = useState("pending");

  const getStatusMessage = () => {
    switch (status) {
      case "pending": return "Processing...";
      case "success": return "Success!";
      case "error": return "Error occurred!";
      default: return "Unknown status";
    }
  };

  return (
    <div>
      <h2>{getStatusMessage()}</h2>

      <button onClick={() => setStatus("pending")}>Pending</button>
      <button onClick={() => setStatus("success")}>Success</button>
      <button onClick={() => setStatus("error")}>Error</button>
    </div>
  );
}

export default StatusExample;

- When to use switch instead of if-else?
Use if-else for different or complex conditions (>, <, &&). 
Use switch when checking one variable against many values


> Conditional Rendering with null:
React allows returning null if we don't want to render anything.

- Example:

function Warning({ show }) {
  if (!show) {
    return null;
  }

  return <h2>Warning Message</h2>;
}

If show is false → nothing is rendered
If show is true → warning message appears
This is useful when a component should hide itself.


> Conditional Rendering of Components:
Sometimes we render different components depending on conditions.

- Example:

function App() {
  const isLoggedIn = true;

  return (
    <div>
      {isLoggedIn ? <Dashboard /> : <Login />}
    </div>
  );
}

This is very common in authentication systems.


> Conditional Rendering Using Functions:
We can also use functions to return different UI.

- Example:

function App() {
  const isAdmin = true;

  function renderContent() {
    if (isAdmin) {
      return <h1>Admin Panel</h1>;
    } else {
      return <h1>User Dashboard</h1>;
    }
  }

  return <div>{renderContent()}</div>;
}

Function decides what UI to render.


> Nested Conditional Rendering
Sometimes conditions can be nested.

- Example:

{user ? (
  user.isAdmin ? <AdminPanel /> : <UserDashboard />
) : (
  <Login />
)}


> Conditional Rendering with Lists:
Example:

const items = ["Apple", "Orange"];

return (
  <div>
    {items.length === 0 ? (
      <p>No items</p>
    ) : (
      <ul>
        {items.map(item => <li>{item}</li>)}
      </ul>
    )}
  </div>
);


> Conditional Rendering with Props:
Example:

function Greeting({ isLoggedIn }) {
  return (
    <h1>
      {isLoggedIn ? "Welcome Back!" : "Please Sign In"}
    </h1>
  );
}
