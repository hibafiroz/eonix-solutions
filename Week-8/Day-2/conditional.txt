> Conditional rendering:

Conditional rendering in React means displaying different UI elements or components based on certain conditions such as state, props, or user actions. 
React uses normal JavaScript conditions like if/else, ternary operators, and logical AND to control what gets rendered in JSX.


Conditional rendering is used when UI should change dynamically depending on data or state.

Common cases:
Showing login/logout buttons
Displaying loading spinner
Showing error messages
Rendering different components
Displaying data only if it exists
Showing admin panel only for admins

 if / else Statement
Used outside JSX because JSX cannot directly contain if statements.
function App() {
  const isLoggedIn = true;

  if (isLoggedIn) {
    return <h1>Welcome User</h1>;
  } else {
    return <h1>Please Login</h1>;
  }
}

React supports multiple ways to handle conditions.

Main methods:
if / else
Element variables
Ternary operator
Logical AND (&&)
Switch statement
Conditional rendering with null
Conditional component rendering
Conditional rendering with functions


Element Variables
Sometimes we store JSX in a variable based on a condition.

Example:

function App() {
  const isLoggedIn = true;
  let message;

  if (isLoggedIn) {
    message = <h1>Welcome back!</h1>;
  } else {
    message = <h1>Please login</h1>;
  }

  return <div>{message}</div>;
}
Explanation:

JSX is stored in message

React renders the variable inside JSX



Ternary Operator (Most Common)
The ternary operator is the most used method for conditional rendering.

Syntax:

condition ? trueResult : falseResult
Example:

function App() {
  const isLoggedIn = true;

  return (
    <div>
      {isLoggedIn ? <h1>Welcome</h1> : <h1>Please Login</h1>}
    </div>
  );
}
Explanation:

If isLoggedIn is true → show Welcome

Otherwise → show Please Login

This is commonly used because it is short and readable.



Logical AND Operator (&&)
Used when you want to render something only if condition is true.

Syntax:

condition && expression
Example:

function App() {
  const hasMessage = true;

  return (
    <div>
      {hasMessage && <p>You have a new message</p>}
    </div>
  );
}
Explanation:

If hasMessage is true → message appears

If false → nothing appears
If the condition is false, React renders nothing.



Conditional Rendering with null
React allows returning null if you don't want to render anything.

Example:

function Warning({ show }) {
  if (!show) {
    return null;
  }

  return <h2>Warning Message</h2>;
}
Explanation:

If show is false → nothing is rendered

If show is true → warning message appears

This is useful when a component should hide itself.



Conditional Rendering of Components
Sometimes we render different components depending on conditions.

Example:

function App() {
  const isLoggedIn = true;

  return (
    <div>
      {isLoggedIn ? <Dashboard /> : <Login />}
    </div>
  );
}
Explanation:

If user logged in → Dashboard component appears

Otherwise → Login component appears

This is very common in authentication systems.



Conditional Rendering Using Functions
You can also use functions to return different UI.

Example:

function App() {
  const isAdmin = true;

  function renderContent() {
    if (isAdmin) {
      return <h1>Admin Panel</h1>;
    } else {
      return <h1>User Dashboard</h1>;
    }
  }

  return <div>{renderContent()}</div>;
}
Explanation:

Function decides what UI to render.

11. Nested Conditional Rendering
Sometimes conditions can be nested.

Example:

{user ? (
  user.isAdmin ? <AdminPanel /> : <UserDashboard />
) : (
  <Login />
)}
Logic:

If user exists
      If admin
            show AdminPanel
      else
            show UserDashboard
Else
      show Login
12. Real World Example (Loading State)
Very common example in applications.

function App() {
  const loading = true;

  return (
    <div>
      {loading ? <p>Loading...</p> : <p>Data Loaded</p>}
    </div>
  );
}
When API is loading → Loading...

After data arrives → Data Loaded

13. Conditional Rendering with Lists
Example:

const items = ["Apple", "Orange"];

return (
  <div>
    {items.length === 0 ? (
      <p>No items</p>
    ) : (
      <ul>
        {items.map(item => <li>{item}</li>)}
      </ul>
    )}
  </div>
);
Explanation:

If list empty → show No items

Otherwise → render list

14. Conditional Rendering with Props
Example:

function Greeting({ isLoggedIn }) {
  return (
    <h1>
      {isLoggedIn ? "Welcome Back!" : "Please Sign In"}
    </h1>
  );
}
Parent component passes isLoggedIn.

Child decides what to render.

15. Best Practices
Recommended practices:

1. Use ternary for simple conditions
Good:

{isLoggedIn ? <Dashboard /> : <Login />}
2. Use && for single condition
Good:

{isAdmin && <AdminPanel />}
3. Avoid deeply nested ternaries
Bad:

condition1 ? condition2 ? A : B : C
Better use:

functions

if statements

4. Keep JSX readable
If logic becomes complex → move it to functions or variables.

16. Interview Definition
If interviewer asks:

What is conditional rendering in React?

Answer:

Conditional rendering in React means displaying different UI elements or components based on certain conditions such as state, props, or user actions. React uses normal JavaScript conditions like if/else, ternary operators, and logical AND to control what gets rendered in JSX.
