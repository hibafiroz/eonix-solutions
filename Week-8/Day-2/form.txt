> form handling basics:

Form Handling in React means managing and processing data from input fields and use it in our application.
using state and event handlers such as onChange and onSubmit.

In normal HTML forms, when we submit a form, the page reloads automatically and the data is sent to the server.
But in React, we usually stop the page reload and handle everything using JavaScript.

Whenever the user types inside an input field:
onChange event runs
We take the value from the input
We store that value in React state

Then when the form is submitted:
onSubmit runs
We process the data

Basic HTML Form (Normal Behavior)
In normal HTML:

<form>
  <input type="text">
  <button type="submit">Submit</button>
</form>

Default behavior:
User submits form
Browser reloads the page
Data is sent to server

But in React:
We prevent page reload
We handle the form with JavaScript

import { useState } from "react";

function App() {

  const [name, setName] = useState("");

  function handleSubmit(e) {
    e.preventDefault();                 //This stops the browser’s default form behavior
    console.log(name);
  }

  return (
    <form onSubmit={handleSubmit}>

      <input
        type="text"
        value={name}
        onChange={(e) => setName(e.target.value)}
      />

      <button type="submit">Submit</button>

    </form>
  );
}


Main Concepts of Form Handling
React form handling mainly uses three things:
State
Event Handlers
Controlled Components



> onChange Event
The onChange event runs whenever the user changes the input value.

onChange={(e) => setName(e.target.value)}
Explanation:
e = event object
e.target = the element that triggered event
e.target.value = current value of input
So React reads the user input and updates state.


3. Form Submission
Forms usually have a submit event.

<form onSubmit={handleSubmit}>

When user clicks submit button or presses Enter, the onSubmit event fires.
handleSubmit Function

Example:

function handleSubmit(e) {
  e.preventDefault();
  console.log("Form Submitted");
}


> Handling Multiple Inputs:

Real forms usually have many inputs like:

name, email, password
Instead of creating many states like this:

name state
email state
password state

we usually keep one state object.

const [formData, setFormData] = useState({
  name: "",
  email: "",
  password: ""
});

Example:

import { useState } from "react";

function Form() {

  const [formData, setFormData] = useState({
    name: "",
    email: "",
    password: ""
  });

  function handleChange(e) {

    const { name, value } = e.target;

    setFormData({
      ...formData,
      [name]: value
    });

  }

  return (
    <form>

      <input
        name="name"
        value={formData.name}
        onChange={handleChange}
      />

      <input
        name="email"
        value={formData.email}
        onChange={handleChange}
      />

      <input
        name="password"
        value={formData.password}
        onChange={handleChange}
      />

    </form>
  );
}
Explanation:

name attribute → identifies input
[value] → gets value from state
handleChange → updates correct property
Key concept:

[name]: value
This updates the correct field dynamically.


Handling Different Input Types
Forms contain many types of inputs.

React handles each slightly differently.

Text Input
<input
  type="text"
  value={name}
  onChange={(e) => setName(e.target.value)}
/>
Textarea
<textarea
  value={message}
  onChange={(e) => setMessage(e.target.value)}
></textarea>
Select Dropdown
<select
  value={country}
  onChange={(e) => setCountry(e.target.value)}
>

  <option value="India">India</option>
  <option value="USA">USA</option>

</select>
Checkbox
Checkbox uses checked property.

Example:

const [isChecked, setIsChecked] = useState(false);

<input
  type="checkbox"
  checked={isChecked}
  onChange={(e) => setIsChecked(e.target.checked)}
/>
Explanation:

e.target.checked → true or false
Radio Buttons
Radio buttons represent one selected option from many.

Example:

const [gender, setGender] = useState("");

<input
  type="radio"
  value="male"
  checked={gender === "male"}
  onChange={(e) => setGender(e.target.value)}
/>

<input
  type="radio"
  value="female"
  checked={gender === "female"}
  onChange={(e) => setGender(e.target.value)}
/>
Resetting a Form
Since inputs are controlled by state, resetting is easy.

Example:

setFormData({
  name: "",
  email: "",
  password: ""
});
All inputs become empty.

Basic Form Validation
React can validate inputs before submitting.

Example:

function handleSubmit(e) {

  e.preventDefault();

  if(name === ""){
    alert("Name is required");
    return;
  }

  console.log(name);
}
Validation types:

Required fields

Email format

Password length

Number range

Pattern matching

Form Handling Flow
Typical React form flow:

User enters input
        ↓
onChange event triggers
        ↓
State updates
        ↓
React re-renders
        ↓
User submits form
        ↓
onSubmit event triggers
        ↓
handleSubmit function runs
        ↓
Data processed or sent to API
Best Practices
1. Use Controlled Components
Always manage form data with React state.

2. Use preventDefault()
Always stop default form reload.

3. Use Meaningful Input Names
name="email"
name="password"
Helps with dynamic handling.

4. Keep State Organized
For large forms, use one object state.

5. Validate Before Submitting
Never submit invalid data.

Common Mistakes
Missing onChange
<input value={name} />
Input becomes read-only.

Forgetting preventDefault()
Page reloads unexpectedly.

Not Using name Attribute
Dynamic input handling will not work.
