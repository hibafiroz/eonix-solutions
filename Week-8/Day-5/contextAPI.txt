> Context API:

Context API is used to share global data like user, theme or authentication status across multiple components without 
passing props manually at every level. t is mainly used to solve the Prop Drilling problem.


> Prop Drilling Problem:

Prop drilling means passing data from a parent component to a deeply nested child component through 
multiple intermediate components, even if those intermediate components don’t use the data.

function App() {
  const user = "Hiba";
  return <Home user={user} />;
}

function Home({ user }) {
  return <Profile user={user} />;
}

function Profile({ user }) {
  return <Settings user={user} />;
}

function Settings({ user }) {
  return <h1>{user}</h1>;
}

Here we passed user through 3 components just to reach Settings.
This is Prop Drilling.


> When to Use Context API:
Use Context when data is global and many components need it:

Logged-in user, Theme (dark/light), Language, Auth status, Global settings
Should not use Context for everything but only for global data.


> Steps to Use Context API (There are 4 steps):

1. Create Context:

import { createContext } from "react";
export const UserContext = createContext();


2. Create Provider:

import { useState } from "react";
import { UserContext } from "./UserContext";

export const UserProvider = ({ children }) => {
  const [user, setUser] = useState("Hiba");

  return (
    <UserContext.Provider value={{ user }}>
      {children}
    </UserContext.Provider>
  );
};


3. Wrap App with Provider:

<UserProvider>
  <App />
</UserProvider>


4. Use Context in Component:

import { useContext } from "react";
import { UserContext } from "./UserContext";

const Profile = () => {
  const { user } = useContext(UserContext);
  return <h1>Hello {user}</h1>;
};


> useContext vs Context API:

Context API is used to create and provide the global data, and 
useContext is a hook used to consume that data inside components.
Context is not the data. Context is the connection between Provider and Consumer.


src
 ├── context
 │     └── UserContext.js
 ├── components
 │     ├── Home.jsx
 │     ├── Profile.jsx
 │     └── Settings.jsx
 └── App.jsx


- Context API vs Redux?
Context API is used for simple global state, while Redux is used for complex state management in large applications.
