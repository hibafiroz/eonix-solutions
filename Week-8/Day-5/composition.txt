> Composition:

Composition in React means building complex UI by combining smaller, reusable components.
Instead of writing one big component, we create small components and put them inside other components using props or children.

In React, we use Composition instead of Inheritance bcz Composition makes components more reusable and flexible.


> Composition happens in three ways:

1. Using children:

function Card({ children }) {
  return (
    <div className='card'>
      {children}
    </div>
  );
}

in app.jsx:
  <Card>
    <h1>Hello Hiba</h1>
  </Card>


- Version 1
function Card(props) {
  return <div className="card">{props.children}</div>;
}

- Version 2 (Destructuring shorter way)
function Card({ children }) {
  return <div className="card">{children}</div>;
}


> Destructuring multiple props + children

function Card({ title, children, className }) {
  return (
    <div className={className}>
      <h2>{title}</h2>
      {children}
    </div>
  );
}

<Card title="Profile" className="card">
  <h1>Hello Hiba</h1>
</Card>


2. Using props:

- Card component:
function Card(props) {
  return (
    <div>
      <h2>{props.title}</h2>
      {props.children}
    </div>
  );
}
export default Card;


- Welcome componenet:
function Welcome(props) {
  return <h1>Hello {props.name}</h1>;
}
export default Welcome;


- App.jsx (parent):
import Card from "./Card";
import Welcome from "./Welcome";

function App() {
  return (
    <div>
      <Card title="User 1">
        <Welcome name="Hiba" />
      </Card>

      <Card title="User 2">
        <p>This is a paragraph</p>
      </Card>

      <Card title="User 3">
        <button>Click Me</button>
      </Card>
    </div>
  );
}
export default App;

Output will be like
<div class="card">
  <h2>User 1</h2>
  <h1>Hello Hiba</h1>
</div>


3. Component Composition:

Passing a component as a prop.

1. Card Component (Reusable Wrapper)
function Card({ children }) {
  return (
    <div>
      {children}
    </div>
  );
}
export default Card;


2. Product Component
function Product({ name, price }) {
  return (
    <>
      <h3>{name}</h3>
      <p>Price: {price}</p>
      <button>Buy Now</button>
    </>
  );
}
export default Product;

> Inheritance vs Composition:

Composition and Inheritance are two ways to reuse code.
Composition means building components by combining smaller components using props and children.
Inheritance means creating a class from another class using extends.
React recommends composition instead of inheritance because composition is more flexible and reusable, 
while inheritance makes components tightly coupled and harder to maintain.
So in React we use composition to reuse UI and logic.

> There are two main composition patterns in React:

| Pattern        | Example                             |
| -------------- | ----------------------------------- |
| Containment    | Card, Modal, Layout (use children)  |
| Specialization | Button, PrimaryButton, DangerButton |

> Containment Example:
Containment is when a component wraps other components using children.

function Card({ children }) {
  return <div className="card">{children}</div>;
}

function App() {
  return (
    <Card>
      <h2>Product</h2>
      <p>Price: 500</p>
      <button>Buy</button>
    </Card>
  );
}

Card is a container
Content inside Card is children
This pattern is called Containment


> Specialization Example:
Specialization means making different types of the same component using props.

function Button({ text, color }) {
  return <button style={{ backgroundColor: color }}>{text}</button>;
}

function App() {
  return (
    <div>
      <Button text="Save" color="green" />
      <Button text="Delete" color="red" />
    </div>
  );
}



> Note:

1. <Card className="card">Hello</Card>
Here className is a prop → props.className

function Card(props) {
  return <div className="card">{props.children}</div>;
}
Here className="card" is just styling the div, not coming from props.


2. <Card title="Hello">Hi</Card>
behind the scenes.
React.createElement(Card, { title: "Hello" }, "Hi");
React.createElement(
  Component,
  propsObject,
  children
)
So React automatically puts the children into the props object with the key name children.
props = {
  title: "Hello",
  children: "Hi"
