> React.memo:

React.memo is a higher-order component that is used to prevent unnecessary re-rendering of a component. 
It re-renders only when its props change by doing a shallow comparison.
It is used for performance optimization.

A Higher Order Component is a function that wraps a component and adds extra functionality like authentication, logging, styling, etc. 
and then returns a new enhanced component. The original component is not modified.

React.memo only prevents re-render caused by props.
If the component has its own state or context changes, it will still re-render.

In React, when a parent component re-renders, all child components also re-render by default. 
React.memo prevents this by re-rendering the child component only if the props have changed. This improves performance.

React.memo uses shallow comparison to check if props have changed. If props are the same, it skips re-rendering.


Without React.memo
function Child({ name }) {
  console.log("Child rendered");
  return <h1>{name}</h1>;
}
Every time parent renders → Child also renders.

With React.memo
const Child = React.memo(({ name }) => {
  console.log("Child rendered");
  return <h1>{name}</h1>;
});
Now Child renders only when name changes.


> Where to AVOID React.memo:

1. If the component is very small and using React.memo here is useless
2. When Props Change Frequently, React.memo cannot stop re-render → No use.
3. When Passing Objects, Arrays, Functions Without Memoizing
creates new object every render → React.memo fails → unnecessary comparison → waste.
4. When Component Re-renders Rarely


> React.memo is often used with:

useCallback → memoize function
useMemo → memoize value


> Smart component vs Dumb component:

Smart components are components that handle logic, state, and data fetching, 
while dumb components are presentational components that only handle UI and receive data via props.

Another name:
Smart Component → Container Component
Dumb Component → Presentational Component


> How do you optimize React performance?
We can optimize React performance using React.memo, useMemo, useCallback, lazy loading, code splitting, and avoiding unnecessary re-renders.

> Why React.memo not working with objects?"

const [firstCount, setFirstCount] = useState(0);
const [secondCount, setSecondCount] = useState(0);

const person = { id: 123, name: "Harry", age: 15 };

<Detail firstCount={firstCount} person={person} />

React.memo does shallow comparison of props.
Objects are compared by reference, not by value.
When we create an object inside a component, a new object is created on every render, so React thinks props changed and re-renders the component.
To fix this, we use useMemo to memoize the object.

const person = useMemo(() => {
  return { id: 123, name: "Harry", age: 15 };
}, []);



> const Person = React.memo(({ name, age }) => {
    console.log("Person component is re-rendered");
    return (
      <div>
        {name}, {age}
      </div>
    );
  },
  (prevProps, nextProps) => {
    return prevProps.name === nextProps.name;
  }
);

Normally React.memo does shallow comparison of all props.
But here, we are giving our own comparison function:

(prevProps, nextProps) => {
  return prevProps.name === nextProps.name;
}

This means:
If name is same → Do NOT re-render
If name is different → Re-render
age change will be ignored

React.memo accepts a second argument which is a custom comparison function. 
It receives previous props and next props. If the function returns true, React will not re-render the component. 
If it returns false, React will re-render the component. 
This is used when we want to control re-rendering manually.
