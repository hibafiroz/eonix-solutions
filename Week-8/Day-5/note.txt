The value prop makes the input controlled by React state, while defaultValue sets only the initial value and the input becomes uncontrolled and managed by the DOM
return (
    <input
      type="text"
      value={name}
      onChange={(e) => setName(e.target.value)}
    />
  );

What happens if we use value without onChange?
Answer:
If we use value without onChange, the input becomes read-only.
Because React is controlling the value but we are not updating the state.


| value                | defaultValue       |
| -------------------- | ------------------ |
| Controlled           | Uncontrolled       |
| React controls input | DOM controls input |
| Needs onChange       | No onChange needed |
| Dynamic              | Only initial value |
| useState             | useRef             |

function App() {
  return <input type="text" defaultValue="Hiba" />;
}



> Lazy initial state:

It is used when calculating the initial state is expensive. 
Instead of passing the value directly to useState, we pass a function that returns the value. 
React will call this function only once during the first render, so the calculation doesn't run again on every re-render

When a component renders for the first time, React needs a starting value for the state.
const [count, setCount] = useState(0);
initial state = 0            //Initial State is Simple here

When Initial State Needs Calculation
const [value, setValue] = useState(calculateValue());

Maybe calculateValue():
runs a heavy calculation
reads from localStorage
processes a large array

function calculateValue() {
  console.log("calculating...");
  return 100;
}
React components re-render many times.
So console becomes:

calculating...
calculating...
calculating...
But React only needs the value the first time.
So this is wasteful.
The Solution (Lazy Initialization)


> Real Practical Example:

Reading localStorage.
Bad way: const [theme, setTheme] = useState(localStorage.getItem("theme"));

Every render React reads localStorage again.

Better way:
const [theme, setTheme] = useState(() => {
  return localStorage.getItem("theme");
});

Now localStorage is read only once.
