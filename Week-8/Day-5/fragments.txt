> Fragments:

Fragment is a wrapper component that groups multiple elements without adding an extra DOM element and it keeps the DOM clean.

In React, a component must return one parent element.
return (
  <div>
    <h1>Hello</h1>
    <p>Welcome</p>
  </div>
);

But this adds an extra div in the DOM, sometimes we don't want that. So we use Fragment.


> There are 3 Ways to Use Fragment:

1. Using <React.Fragment>:

function App() {
  return (
    <React.Fragment>
      <h1>Hello</h1>
      <p>Welcome</p>
    </React.Fragment>
  );
}


2. Using <Fragment>

function App() {
  return (
    <Fragment>
      <h1>Hello</h1>
      <p>Welcome</p>
    </Fragment>
  );
}


3. Using <> </> (Shortcut) — Most Used
function App() {
  return (
    <>
      <h1>Hello</h1>
      <p>Welcome</p>
    </>
  );
}


| div                    | Fragment          |
| ---------------------- | ----------------- |
| Adds extra DOM element | No extra DOM      |
| Can style              | Cannot style      |
| Used for layout        | Used for grouping |


> When rendering a list, Fragment can accept key, but the shortcut <> </> cannot.

const items = ["Apple", "Banana", "Mango"];

function App() {
  return items.map((item, index) => (
    <Fragment key={index}>
      <h1>{item}</h1>
      <p>Fruit</p>
    </Fragment>
  ));
}


> Use Fragment when:

- Returning multiple elements
- Avoiding extra div in DOM
- Table structure


function Table() {
  return (
    <table>
      <tr>
        <>
          <td>One</td>
          <td>Two</td>
        </>
      </tr>
    </table>
  );
}

In elements like table, tr, and ul, we cannot place a div inside them because it breaks the HTML structure. 
In such cases, we use React Fragment to group elements without adding an extra DOM node.
