> Fragments:

Fragment is a wrapper component in React that groups multiple elements without adding an extra DOM node. 
It helps keep the DOM clean and is useful when returning multiple elements from a component

In React, a component must return one parent element.
return (
  <div>
    <h1>Hello</h1>
    <p>Welcome</p>
  </div>
);

But this adds an extra div in the DOM, sometimes we don’t want that. So we use Fragment.


> There are 3 Ways to Use Fragment:

1. Using <React.Fragment>:

import React from "react";

function App() {
  return (
    <React.Fragment>
      <h1>Hello</h1>
      <p>Welcome</p>
    </React.Fragment>
  );
}


2. Using <Fragment>
import { Fragment } from "react";

function App() {
  return (
    <Fragment>
      <h1>Hello</h1>
      <p>Welcome</p>
    </Fragment>
  );
}


3. Using <> </> (Shortcut) — Most Used
function App() {
  return (
    <>
      <h1>Hello</h1>
      <p>Welcome</p>
    </>
  );
}


| div                    | Fragment          |
| ---------------------- | ----------------- |
| Adds extra DOM element | No extra DOM      |
| Can style              | Cannot style      |
| Used for layout        | Used for grouping |


When rendering a list, Fragment can accept key, but the shortcut <> </> cannot.
import { Fragment } from "react";

const items = ["Apple", "Banana", "Mango"];

function App() {
  return items.map((item, index) => (
    <Fragment key={index}>
      <h1>{item}</h1>
      <p>Fruit</p>
    </Fragment>
  ));
}

Shortcut will NOT work here. We use <Fragment> instead of <> when we need to pass a key attribute


> Use Fragment when:

Returning multiple elements
Avoiding extra div in DOM
Table structure

function Table() {
  return (
    <table>
      <tr>
        <>
          <td>One</td>
          <td>Two</td>
        </>
      </tr>
    </table>
  );
}
If you use <div> inside <tr>, it breaks HTML structure. So we use Fragment.
