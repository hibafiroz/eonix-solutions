> React Router DOM:

React Router DOM is a library used for navigation in React applications.
Since React is a single-page application, it does not reload the entire page when navigating.
Instead, React Router DOM updates the URL and renders different components based on the route.

We usually wrap the whole app inside BrowserRouter, then inside that we define Routes, 
and inside Routes we define each Route with a path and the component to show.


1. Install
npm install react-router-dom

2. Basic Setup
import { BrowserRouter, Routes, Route } from "react-router-dom";
import Home from "./Home";
import About from "./About";

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}

| Tag           | Purpose          |
| ------------- | ---------------- |
| BrowserRouter | Wraps whole app  |
| Routes        | Holds all routes |
| Route         | Single route     |
| path          | URL              |
| element       | Component        |

> BrowserRouter:

BrowserRouter is used to wrap the entire React application so that routing can work. 
It connects the React app with the browser URL.
Because of BrowserRouter, when the URL changes, 
React will render a different component without reloading the page.
So it basically enables single page application routing in React.


> We Use it in App.js.

import { BrowserRouter } from "react-router-dom";
import App from "./App";

function Root() {
  return (
    <BrowserRouter>
      <App />
    </BrowserRouter>
  );
}

| Component     | Job              |
| ------------- | ---------------- |
| BrowserRouter | Enables routing  |
| Routes        | Holds all routes |
| Route         | One route        |
| Link          | Navigation       |
| useNavigate   | Redirect         |
| useParams     | Get URL data     |



> Routes & Route:

Routes is a container that holds all the Route components. 
It checks the current URL and renders the matching Route.

Route is used to define a single path and which component should be displayed for that path.

In React Router v6:

Old version:

<Route path="/" component={Home} />
New version:

<Route path="/" element={<Home />} />



> useParams:

useParams is a React Router hook used to extract dynamic values from the URL, such as an ID,
 which can be used to fetch and display specific data.


Step 1 — Create Dynamic Route
<Route path="/user/:id" element={<User />} />
:id → dynamic value

Step 2 — Get ID in User Component
import { useParams } from "react-router-dom";

function User() {
  const { id } = useParams();

  return <h1>User ID: {id}</h1>;
}


For example, in a project, when I click a user from the user list, 
I navigate to /user/ID. Then in the user details page, 
I use useParams to get the ID from the URL, and using that ID 
I fetch the user data from the backend and display it.



> useNavigate:

useNavigate is a React Router hook used to redirect users to another route programmatically based on some action like login, logout, or form submission.

import { useNavigate } from "react-router-dom";

function Login() {
  const navigate = useNavigate();

  const handleLogin = () => {
    // login logic
    navigate("/dashboard");
  };

  return <button onClick={handleLogin}>Login</button>;
}

Go Back Example
<button onClick={() => navigate(-1)}>Go Back</button>


> Nested Routes:

Nested routing is used when we have a parent page like dashboard and inside that we have child pages like profile, settings, orders etc. 
In React Router, we define a Route inside another Route, and we use Outlet in the parent component to render the child components.

| Page      | Route               |
| --------- | ------------------- |
| Dashboard | /dashboard          |
| Profile   | /dashboard/profile  |
| Orders    | /dashboard/orders   |
| Settings  | /dashboard/settings |
This is exactly where nested routes are used.



> Protected Route:

Protected routes are used to restrict access to certain pages like dashboard or profile so that only authenticated users can access them

<Route
  path="/dashboard"
  element={
    <ProtectedRoute>
      <Dashboard />
    </ProtectedRoute>
  }
/>
Now: Logged in → Dashboard
     Not logged in → Login page



> 404 page:

In React Router, we create a 404 page using a wildcard route with path "*". If the user enters a URL that doesn't match any existing route, the NotFound component will be displayed.

| URL              | Page            |
| ---------------- | --------------- |
| /                | Home            |
| /login           | Login           |
| /dashboard       | Dashboard       |
| /product/101     | Product Details |
| /something-wrong | 404 Page        |

import { useNavigate } from "react-router-dom";

function NotFound() {
  const navigate = useNavigate();

  return (
    <>
      <h1>404 - Page Not Found</h1>
      <button onClick={() => navigate("/")}>Go Home</button>
    </>
  );
}
