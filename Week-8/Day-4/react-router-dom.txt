> React Router DOM:

React Router DOM is a library used for navigation in React applications.
Since React is a single-page application, we dont reload the entire page when navigating between pages. 
Instead, React Router changes the URL and renders different components based on the route.

We usually wrap the entire application inside BrowserRouter so that routing can work in the app. 
Inside BrowserRouter, we use Routes to define all the available routes. Then inside Routes, 
we define multiple Route components, where each Route has a path and an element (component) to display.


1. Install
npm install react-router-dom


2. Basic Setup
import { BrowserRouter, Routes, Route } from "react-router-dom";
import Home from "./Home";
import About from "./About";

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}



> BrowserRouter:

BrowserRouter is used to wrap the entire React application so that routing can work. 
It connects the React app with the browser URL.
Because of BrowserRouter, when the URL changes, React will render a different component without reloading the page.
So it basically enables single page application routing in React.


> We Use it in Main.js.

import { BrowserRouter } from "react-router-dom";
import App from "./App";

function Root() {
  return (
    <BrowserRouter>
      <App />
    </BrowserRouter>
  );
}

| Component     | Job              |
| ------------- | ---------------- |
| BrowserRouter | Enables routing  |
| Routes        | Holds all routes |
| Route         | One route        |
| Link          | Navigation       |
| useNavigate   | Redirect         |
| useParams     | Get URL data     |



> Routes & Route:

- Routes is a container that holds all the Route components. 
- Route is used to define a single path and which component should be displayed for that path.

In React Router v6:
<Route path="/" component={Home} />

New version:
<Route path="/" element={<Home />} />



> Link:

Link is used to navigate between pages without reloading the page.

import { Link } from "react-router-dom";

<Link to="/">Home</Link>
<Link to="/about">About</Link>

Why not <a> tag?
Because <a> reloads the page, but Link does not.


> NavLink:

NavLink is used for navigation and it automatically applies active class styling to the current route.
It checks whether the current URL matches the link path. If it matches, it sets the link as active and 
we can apply active class or active styling using className or style based on the isActive property.

<NavLink to="/about" style={({ isActive }) => ({
    color: isActive ? "red" : "black"
  })}
>
  About
</NavLink>



> Navigate Component:

Navigate is used to redirect from one path to another path.

Instead of this    <Route path="/about" element={<About />} />
We write like      <Route path="/home" element={<Navigate to="/" />} />

Navigate is used for redirection, while Route is used to render a component for a specific path.
