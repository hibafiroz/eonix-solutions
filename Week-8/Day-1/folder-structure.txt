> React Project Folder Structure:

After creating the project, we will see something like this:

my-react-app
│
├─ node_modules
├─ public
├─ src
│   ├─ App.jsx
│   ├─ main.jsx
│   ├─ index.css
│
├─ package.json
├─ vite.config.js

- node_modules:
This folder contains all installed packages.
Example:
React
dependencies
libraries
It is automatically created when you run:
npm install

- public
This folder contains static files.
Example:
images
icons
static assets
Files inside public are served directly to the browser.

- src (Most Important Folder)
This is where all your React code lives.
Inside this folder you will write:
components
styles
logic
UI

- main.jsx
This is the entry point of the React app.
It connects React with the HTML page.
Example:

import React from "react"
import ReactDOM from "react-dom/client"
import App from "./App"
ReactDOM.createRoot(document.getElementById("root")).render(
  <App />
)
Here React renders the App component.

- App.jsx
This is the main component of your application.
Example:
function App() {
  return <h1>Hello React</h1>
}
export default App
This component controls the main UI.

- package.json
This file contains:
project information
installed dependencies
scripts

Example scripts:

npm run dev
npm run build


As projects grow, developers organize files like this:

src
│
├─ components
│   ├─ Navbar.jsx
│   ├─ StudentCard.jsx
│
├─ pages
│   ├─ Home.jsx
│   ├─ About.jsx
│
├─ App.jsx
├─ main.jsx


> Explain the React project folder structure.

In a React project created using Vite, the main folders include node_modules, public, and src. 
The node_modules folder stores all installed dependencies. 
The public folder contains static assets like images. 
The src folder is the most important because it contains the application code, including components and styles. 
Inside src, main.jsx is the entry point that renders the App component, and App.jsx usually acts as the main component of the application.
